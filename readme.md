# 随声提词

一款面向口播、直播、短视频录制场景的微信小程序提词器。当前版本支持基础匀速提词、语音智能跟随、横竖屏适配、录制模式、台本整理和本地历史记录。

## 核心能力

- 基础提词：按每分钟字数匀速滚动，支持开始倒计时。
- 智能跟随：接入微信同声传译插件，将朗读内容与台本做本地匹配，识别成功后自动推进到下一行。
- 录制模式：在提词页开启相机背景，可开始录制、保存到相册，并支持前后摄像头切换。
- 横竖屏适配：进入提词页时可选择自动、竖屏或横屏；横屏下提词区、设置面板和录制按钮有独立布局。
- 显示设置：支持字号、行距、字距、对齐方式、基准线位置、聚焦模式、文字颜色和背景颜色。
- 滚动设置：基础模式支持慢、标准、快三档预设，也可用滑杆细调速度。
- 台本整理：合并复制粘贴产生的碎换行，保留段落，并按句末标点重新断行。
- 最近台本：首页自动保留最近台本，支持一键载入和删除。
- 新手引导：首次进入提词页展示基础操作说明。
- 本地保存：台本历史、显示设置和引导状态都保存在本地缓存。

## 页面说明

### 首页 `pages/index/index`

首页用于输入或粘贴台本，支持：

- 调整台本输入框高度
- 整理、清空台本
- 选择提词方向：自动、竖屏、横屏
- 开启或关闭录制模式
- 保存台本到最近记录
- 进入提词页
- 查看、载入、删除最近台本

### 编辑页 `pages/editor/editor`

编辑页保留了脚本标题和正文编辑能力，支持保存、整理和清空内容。当前首页主流程使用“最近台本”，编辑页仍可用于独立脚本维护。

### 提词页 `pages/prompter/prompter`

提词页是核心使用界面，支持：

- 基础模式：根据字数和速度计算滚动动画时长
- 跟随模式：监听语音识别结果并匹配台本位置
- 手动拖动台本位置
- 点击屏幕暂停
- 运行中显示“提词中”或“聆听中”状态
- 长时间未操作自动隐藏顶部和底部操作区
- 设置面板分为显示、滚动、颜色、高级四组
- 录制模式下显示相机画面和录制控制

## 智能跟随逻辑

智能跟随由 `miniprogram/utils/matcher.js` 中的 `SmartMatcher` 负责。核心策略是：

1. 先将台本和识别文本规范化，去掉标点、空格和大小写差异。
2. 只处理满足最小长度的增量识别文本，降低误触发。
3. 在当前位置附近窗口内搜索最佳匹配。
4. 对匹配长度和位置合理性计算置信度。
5. 只有置信度达标才推进位置，未匹配时不自动滚动。
6. 提词页只允许智能跟随向前推进，避免识别抖动导致上下跳动。

当前跟随滚动会按行高吸附，并让正在朗读的内容落在基准线下方第一行附近。

## 运行方式

1. 使用微信开发者工具打开项目根目录。
2. 确认小程序根目录为 `miniprogram/`。
3. 在微信开发者工具中编译运行。
4. 如需测试智能跟随，请在真机或支持录音能力的环境中授权麦克风。
5. 如需测试录制模式，请授权相机、麦克风和保存到相册权限。

## 依赖与权限

项目使用微信小程序原生能力开发，没有常规 npm 构建脚本。

智能跟随依赖微信同声传译插件：

```json
{
  "plugins": {
    "wechat-si": {
      "version": "0.3.5",
      "provider": "wx069ba97219f66d99"
    }
  }
}
```

涉及的小程序能力包括：

- `scope.record`：语音识别和录制需要麦克风权限
- `scope.camera`：录制模式需要相机权限
- 保存视频到相册：结束录制后保存视频文件

## 本地缓存

缓存键集中定义在 `miniprogram/utils/helpers.js`：

- `script_history`：首页最近台本
- `scripts`：编辑页脚本列表
- `prompter_settings`：提词显示和滚动设置
- `temp_prompter_script`：进入提词页时临时传递长台本
- `hasSeenGuide`：新手引导是否已看过

## 当前限制

- 智能跟随依赖微信 ASR 返回结果，首次识别和长语音分段可能存在轻微延迟。
- 录制暂停目前使用停止录制保存的方式处理，因为小程序相机录制 API 不提供真正的暂停和继续。
- 台本整理是规则化处理，不会改写文案语义。
- 数据默认保存在本地缓存，卸载小程序或清理缓存后可能丢失。

## 后续优化方向

- 增加云端或文件导入导出，降低本地缓存丢失风险。
- 为智能跟随加入更细的识别延迟补偿参数。
- 增加更多适合录制场景的颜色主题。
- 给编辑页和首页历史记录建立更统一的脚本管理入口。
